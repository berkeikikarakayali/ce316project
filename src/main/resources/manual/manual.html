Integrated Assignment Environment (IAE)

The IAE helps lecturers evaluate batches of student ZIP submissions locally. Each assignment lives in a single SQLite project file (.iae). No database server is required.

1. Workflow

  1. Create reusable language configurations (compiler/interpreter paths + arguments).
  2. Create/open a project (wizard tab) selecting configuration, ZIP folder, expected output file, argv extras, timeouts, normalization.
  3. Place student archives named <studentId>.zip inside the ZIP folder.
  4. Choose Run → Execute assignment. The tool extracts archives with pure Java ZIP APIs, compiles/runs each submission, compares stdout with your expected output file, and persists reports.
  5. Open results anytime via File → Open project.

2. Configuration placeholders

Compile/run argument templates support tokens:

  • {SRC} — absolute path to the discovered student source file.
  • {DIR} — absolute working directory for the extracted submission.
  • {BASE} — filename portion of the student source.

Example C workflow: 

compiler_path = /usr/bin/gcc
compile_args  = ["-o","main","{SRC}"]
run_args      = ["./main"]

3. Project argv extras

Extra arguments appended after configuration run tokens support JSON arrays such as ["red","blue"] or legacy whitespace-separated tokens like case1 case2.

4. Normalization modes

  • STRICT — unify CRLF/LF then compare verbatim.
  • TRIM_WHITESPACE — trim each line before comparing.
  • CASE_INSENSITIVE — trim lines then lowercase entire buffers.

5. Packaging

Bundle JavaFX via jpackage on Windows and wrap the output folder with Inno Setup to satisfy course installer requirements. Keep lecturers’ .iae databases outside Program Files so uninstall does not erase grading history.
